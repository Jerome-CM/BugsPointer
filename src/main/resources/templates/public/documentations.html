Intégration

Une vidéo tuto est bientôt prévue pour cette section

Télécharger les fichiers

Pour télécharger le projet rendez-vous sur l’url : https://www.bugspointer.com/download
Vous devez avoir un compte valide pour obtenir les différents fichiers

Implémenter les fichiers sur votre site

Pour profiter de BugsPointer sur votre site, il vous suffit de copier/coller les fichiers que vous avez téléchargés.
Placez-les dans votre arborescence :

lien.html
bouton.html
modalStyle.css a placé dans votre dossier de style
modalControl.js a placé dans votre dossier javascript
modal.html a placé à la racine de votre site

Pour que les fichiers soient visibles sur votre page, vous devez ensuite :

  • inclure un lien et/ou un bouton sur la page à débugger
  • inclure le fichier modalStyle.css et modalControl.js dans votre balise <head></head>
  • inclure en fin de page le fichier modal.html

L'intégration des fichiers est différente selon le langage utilisé : Voir des exemples

Configuration

Une vidéo tuto est bientôt prévue pour cette section

Renseigner votre clé public

Rendez-vous dans votre page Account pour trouver votre Public Key.

Coller là dans le fichier modalControl.js à la ligne 1, à la place de YOUR_KEY_HERE :

1| const key = "YOUR_KEY_HERE"; /* Ajoutez votre clé publique ici, entre les "" */

Renseigner votre domaine

Rendez-vous dans votre page Account et assurez-vous d'avoir correctement renseigné la partie " Domaine "

Vous pouvez le vérifier ici ( si vous êtes connecté ) : https://www.bugspointer.com/app/private/account#domain

L'intégration de votre clé publique et de votre nom de domaine est obligatoire pour recevoir vos rapports de bugs

Customisation

Une vidéo tuto est bientôt prévue pour cette section

Lien sur votre page

La customisation du lien contenant le formulaire n'appartient qu'à vous. Vous êtes libre d'appliquer le style ainsi que le message que vous souhaitez sur ces boutons


<a class="bugspointer-open-popup" id="bugspointer_open_popup">Signaler un bug</a>

Cependant, vous pouvez ajouter n'importe quelles classes en plus de bugspointer-open-popup mais, vous ne devait en aucun cas toucher à l'id

Logo sur votre page

La customisation du logo est également possible. Pour cela, rendez-vous dans le fichier modalStyle.css


Les couleurs et polices sont changeables dans la classe :root, cependant comme conformément à nos cgu, vous ne pouvez pas changer les ids et les classes du fichier modal.html et modalStyle.css, mais vous pouvez ajouter d'autres classes afin de parfaire le style, selon votre charte graphique.

Changer les classes et les ids originales nuira au bon fonctionnement de l'application

Comment déclarer un bug

En tant qu'utilisateur d'un site internet possédant la technologie de BugsPointer, vous pouvez signaler un bug via un lien sur la page, ou via un logo visible sur la page. Le propriétaire du site est libre de choisir quelle méthode il souhaite utiliser.


En cliquant sur n'importe lequel des moyens, une fenêtre apparaît sur votre navigateur.

Cette fenêtre est constituée de l'url de la page sur laquelle vous êtes, un bouton permettant de pointer l'endroit de la page où le bug se situe et une description

Pointer un bug, ainsi qu'une description de minimum 10 caractères est nécessaire pour l'envoi d'un rapport de bug

Gestion des bugs

Une vidéo tuto est bientôt prévue pour cette section

Nouveau bug

Lors du signalement d'un bug sur votre site par votre utilisateur, un nouveau bug vous est ajouté dans votre Dashboard.
Suivant vos préférences de notifications, vous allez ( ou non ) recevoir un mail à la réception du bug.
Vous le retrouverez ensuite dans la tuile rouge "new bug".

Une fois sur la page du bug, vous pouvez le classer en 3 catégories :

  • En cours
  • Résolu
  • Ignoré

Bug en cours

Lorsque vous classez un bug dans la catégorie "en cours", il lui est ajouté l'heure à laquelle vous l'avez vu afin de confirmer son déplacement dans cette catégorie.

Corriger ce bug et confirmer sa résolution

Bug résolu

Une fois un bug résolu, il sera déplacé dans cette catégorie afin de servir d'archive pour garder une trace de tous les bugs reçus dans votre Dashboard.

Toutefois, si vous vous apercevez que le bug n'est pas résolu, vous pouvez le réouvrir, ce qui le placera automatiquement dans la catégorie "en cours".

Bug ignoré

Si un utilisateur vous a signalé un bug que vous connaissiez déjà, ou bien que ce n'est pas un bug pour vous, vous pouvez ignorer ce bug.

Comme un bug résolu, il sera gardé en archive.
Toutefois, si vous jugez que ce n'est plus un bug à ignorer, vous pouvez annuler l'ignorance, dans ce cas, le bug sera reclassé dans la catégorie "en cours".
